# 우선순위 Spec

## 작업 우선순위

### 🔴 Critical (즉시)
**데이터 손실 위험**
- 손상된 YAML
- 깨진 Wikilink (404)
- 중복 파일명

### 🟡 High (당일)
**시스템 일관성**
- 고립 노트 (백링크 0개)
- 핵심개념 100줄 초과
- 출처 미기입

### 🟢 Medium (주간)
**품질 개선**
- 태그 정규화
- MOC 갱신
- 즉흥메모 pending 처리

### ⚪ Low (월간)
**최적화**
- 파일명 개선
- 오래된 보관 검토
- 중복 개념 통합

## 자동화 범위

### ✅ 자동 실행
- YAML 메타데이터 추가
- 파일명 정규화
- 태그 정규화
- 백링크 검증
- 즉흥메모 저장

### ⚠️ 사용자 확인 필요
- 노트 삭제/병합
- 핵심개념 생성
- MOC 생성/갱신
- 프로젝트 폴더 생성
- 100줄 초과 분리
- 보관 처리

**원칙**: 데이터 손실 가능성 시 반드시 확인
