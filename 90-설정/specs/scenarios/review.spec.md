# 리뷰 시나리오 Spec

## 트리거 키워드
- "주간 리뷰", "검증해줘"
- "즉흥메모 리뷰", "링크 점검"

## 리뷰 타입별 프로세스

### 1. 주간 리뷰 (매주 금요일)
```tasks
path includes 10-수집/즉흥메모
type = todo
(completed after 1 week ago) OR (not done)
```

**점검 항목:**
- pending 즉흥메모 처리 제안
- 완료된 TODO 보관 확인
- 고립 노트 검색
- 깨진 Wikilink 목록

### 2. 월간 리뷰 (매월 마지막 일요일)
- 프로젝트 진행률 보고
- 오래된 자료정리 → 핵심개념 제안
- MOC 갱신 필요 항목
- 태그 정규화 일괄 처리

### 3. 분기 리뷰 (3개월마다)
- 보관 대상 검토 (1년 이상 미사용)
- 태그 체계 전체 정리
- 파일명 일괄 정규화
- 중복 개념 통합 제안

## 자동 검증 체크리스트

### 🔴 Critical (즉시)
- [ ] 손상된 YAML
- [ ] 깨진 Wikilink (404)
- [ ] 중복 파일명

### 🟡 High (당일)
- [ ] 고립 노트 (백링크 0개)
- [ ] 핵심개념 100줄 초과
- [ ] 출처 미기입

### 🟢 Medium (주간)
- [ ] 태그 정규화
- [ ] MOC 갱신
- [ ] 즉흥메모 pending 처리

### ⚪ Low (월간)
- [ ] 파일명 개선
- [ ] 오래된 보관 검토

## 리포트 형식
```markdown
## 📊 주간 리뷰 리포트

### 🔴 긴급 이슈
- 깨진 링크: X개
- 출처 없는 핵심개념: Y개

### 🟡 처리 필요
- 고립 노트: Z개
- 오래된 즉흥메모: W개

### ✅ 완료된 작업
- 처리된 즉흥메모: N개
- 생성된 핵심개념: M개
```

## 검증 우선순위
1. 데이터 손실 위험 (Critical)
2. 시스템 일관성 (High)
3. 품질 개선 (Medium)
4. 최적화 (Low)
